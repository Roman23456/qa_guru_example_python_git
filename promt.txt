Проект, направленный на тестирование WEB-приложений.
Общие требования:

 Не менее 7-ми тестов.
 Должно быть несколько тестовых модулей, разделение которых происходит по бизнес-логике продукта. Например, тесты на авторизацию (test_login) и тесты на поиск (search_test).
 Действия, выполняемые в ходе теста, должны логироваться - красиво отображаться в Allure отчете. Для этого можно пользоваться либо аннотацией @allure.step(), либо же конструкцией with allure.step():
 Очень круто, если на уровне тестовых классов будут использоваться аннотации @allure.epic(), @allure.story(), @allure.feature, которые позволяют группировать тесты по категориям в Allure отчете.
 А на уровне тестовых методов - аннотации @allure.tag(), @allure.label(), @allure.severity().
Рекомендуется использование файла conftest.py:

 Файл conftest.py в Python обычно используется для настройки и предоставления общих ресурсов и функциональности для модульных тестов или тестовых сценариев.
 Минимально должен содержать фикстуры @pytest.fixture в которых может производиться конфигурация запуска тестов:
 Базовый URL - browser.config.base_url
 Браузер - browser.driver.name
 Размер браузера - browser.driver.set_window_size
 Удаленный URL от Selenoid - webdriver.Remote
 Ключевое слово yield после которого производится добавление attachments (вложения) в Allure отчет:
 Скриншот страницы.
 HTML страницы (по желанию. На практике - довольно бесполезный кейс).
 Логи браузера.
 Видео прохождения теста.
 Также здесь желательно закрывать браузер browser.close(), чтобы новый тест запустился в новом “чистом” браузере.
Требования к модулям, реализующим паттерн PageObject:

 Класс для PageObject должен лежать в отдельном модуле в выделенном пакете внутри проекта.
 Для реализации шаблона PageObject допускается использовать два варианта:
 Отдельные шаги - заполнение данных о пользователе и подтверждение результата проделанной работы (как, например, подтверждение, что регистрация прошла успешно)
 Все элементы выносить в отдельные поля объекта класса не обязательно, но стоит это сделать с теми элементами, которые будут повторяться.

 Только высокоуровневые шаги - отправить форму с данными или провести регистрацию через форму. Также шагом считаем подтверждение результата проделанной работы (как например, подтверждение, что регистрация прошла успешно).
